# E-Commerce Test Automation Framework\n\nA comprehensive test automation framework for ecommerce applications using **Playwright**, **TypeScript**, and **Page Object Model (POM)** design pattern.\n\n## 🎯 Features\n\n- ✅ **Playwright** - Modern, fast, and reliable E2E testing\n- 🏗️ **Page Object Model** - Maintainable and scalable test structure\n- 🔄 **Data-Driven Testing** - Test multiple scenarios with different data\n- 📊 **Allure Reports** - Beautiful test reporting with screenshots\n- 🔗 **API Testing** - Built-in REST API testing\n- 🌐 **Cross-Browser** - Chrome, Firefox, Safari support\n- 📱 **Mobile Emulation** - Test mobile viewports\n- 🔐 **CI/CD Ready** - GitHub Actions integration\n- 🎨 **TypeScript** - Type safety and better IDE support\n\n## 📋 Prerequisites\n\n- Node.js 16+\n- npm or yarn\n- Git\n\n## 🚀 Installation\n\n```bash\n# Clone the repository\ngit clone https://github.com/vishnu3948/ecommerce-test-automation.git\ncd ecommerce-test-automation\n\n# Install dependencies\nnpm install\n\n# Install Playwright browsers\nnpx playwright install\n```\n\n## 🧪 Running Tests\n\n```bash\n# Run all tests\nnpm run test\n\n# Run tests in headed mode\nnpm run test:headed\n\n# Run specific test type\nnpm run test:smoke\nnpm run test:regression\nnpm run test:cart\nnpm run test:checkout\nnpm run test:integration\nnpm run test:mobile\n\n# Run with UI mode\nnpm run test:ui\n\n# Run in debug mode\nnpm run test:debug\n```\n\n## 📊 Reporting\n\n```bash\n# Generate HTML report\nnpm run test:report\n\n# Generate Allure report\nnpm run test:allure\n\n# View Allure report\nnpm run allure:report\n```\n\n## 🔑 Configuration\n\nCreate a `.env` file based on `.env.example`:\n\n```bash\nBASE_URL=https://your-ecommerce.com\nAPI_BASE_URL=https://api.your-ecommerce.com\nTEST_USER_EMAIL=testuser@example.com\nTEST_USER_PASSWORD=testpassword123\n```\n\n## 📁 Project Structure\n\n```\n├── pages/                # Page Object Model classes\n├── api/                  # API request classes\n├── fixtures/             # Custom fixtures\n├── tests/\n│   ├── e2e/             # UI/E2E tests\n│   ├── api/             # API tests\n│   ├── integration/      # Full flow tests\n│   └── mobile/          # Mobile tests\n├── utils/               # Helper utilities\n└── .github/workflows/   # CI/CD configuration\n```\n\n## 🛠️ Best Practices\n\n1. Use Page Object Model for UI tests\n2. Use data fixtures for test data\n3. Tag tests appropriately (@smoke, @regression, @sanity)\n4. Keep tests independent and parallel-safe\n5. Use meaningful assertions\n6. Capture screenshots on failure\n\n## 📚 Documentation\n\n- [Playwright Docs](https://playwright.dev)\n- [Contributing Guidelines](./CONTRIBUTING.md)\n\n## 📄 License\n\nMIT License\n\n## 💬 Support\n\nFor issues and questions, please create a GitHub issue.\n"
